## Project Overview

NYSDS (New York State Design System) — a Lit 3 web components monorepo providing accessible (WCAG 2.2 AA) UI components, design tokens, and styles for NYS digital products. All components are prefixed with `nys-`.

## Commands

### Build
- `npm run build` — Root build: `tsc --emitDeclarationOnly && vite build`
- `npm run build:packages` — Build all packages in dependency order
- `npm run build:all` — Full production build (clean, lint, build, analyze, manifests)

### Test
- `npm run test` — Web Test Runner across 6 browser targets (Playwright)
- `npm run test:build` — Full build then test
- Per-package: `cd packages/nys-<name> && npm test`

### Lint & Dev
- `npm run lint` / `npm run lint:fix` — ESLint + Stylelint
- `npm run storybook` — Storybook on port 6006
- `npm run gen` — Scaffold a new component

## Architecture

### Monorepo Layout
- `packages/` — workspace packages (tokens, styles, nys-* components, mcp-server)
- `src/index.ts` — Barrel export of all components

### Foundation Packages (built first)
- **tokens** — Design tokens (DTCG format), outputs CSS variables
- **styles** — CSS variables, utility classes, themes; exports `nysds.min.css`

### Component Package Structure
Each `packages/nys-<name>/` contains:
- `src/nys-<name>.ts` — LitElement component
- `src/nys-<name>.scss` — Styles (CSS custom properties)
- `src/nys-<name>.test.ts` — Tests (Mocha, @open-wc/testing)
- `src/nys-<name>.stories.ts` — Storybook stories
- `vite.config.js` — Extends root `defaultConfig`

### Build Pipeline
Each package: `tsc --emitDeclarationOnly` → `vite build`. Package vite configs inherit `emptyOutDir: false` to preserve .d.ts files.

## Key Conventions

- Components extend `LitElement` from Lit 3
- Styles use design tokens from `@nysds/tokens`
- TypeScript strict mode
- CI runs on PRs to develop/main

## MCP Server

Located at `packages/mcp-server/`. Exposes components, tokens, and docs to AI assistants.

- **Package:** `@nysds/mcp-server`
- **Data source:** `dist/custom-elements.json` (generated by `npm run cem`)
- **Build:** `npm run build --workspace=packages/mcp-server`
- **Test:** `npx @modelcontextprotocol/inspector packages/mcp-server/dist/index.js`
- **Implementation plan:** See `PRD.md`
