## Options

### Striped
Adding the `striped` property can improve readability by visually grouping row content with alternating background colors.
<Canvas of={NysTableStories.Striped} />

### Bordered
Adding the `bordered` property can improve readability by adding a line between cells.
<Canvas of={NysTableStories.Bordered} />

### Sortable
Adding the `sortable` property allows for the table to be reordered in accending or decending order.
<Canvas of={NysTableStories.Sortable} />

### Downloadable
Adding the `download` property adds a button which downloads the data that is being displayed. The `download` property takes in a string that points to the dataset. It does not parse and download the content in the `nys-table`.
<Canvas of={NysTableStories.Downloadable} />

## Events

The `nys-table` component emits the following events:

1. `nys-click`: Emitted when the table is clicked.
2. `nys-focus`: Emitted when the table receives focus.
3. `nys-blur`: Emitted when the table loses focus.

**Event details** \
The eventâ€™s `detail` object includes:
- id (string): The id of the table.
- checked (boolean): Whether the table is currently checked.

You can listen to these events using JavaScript:

```js
// Select the table component
const table = document.querySelector("nys-table");

// Listen for the 'nys-click' event
table.addEventListener("nys-click", (event) => {
  const {id, checked} = event.detail;
	console.log("Table (${id}) is clicked. Checked: ${checked}");
});

// Listen for the 'nys-focus' event
table.addEventListener("nys-focus", () => {
	console.log("Table is focused");
});

// Listen for the 'nys-blur' event
table.addEventListener("nys-blur", () => {
	console.log("Table lost focus");
});
```
