# Dropdownmenu

The **`nys-dropdownmenu`** component display a list of actions that are triggered by a button or interactive element. They help conserve space while grouping related actions.

If an id is not passed, a unique id will be generated.

## Usage

### When to use this component
- To display a short list of related actions
- For account menus and user actions
- For overflow or contextual actions
- When actions are secondary to page content

### When to consider something else
- When presenting a large list of selectable options
- When content includes form inputs or complex layouts
- When actions should remain permanently visible
- When building primary navigation

## Accessibility

The `nys-dropdownmenu` component includes the following accessibility-focused features:

- Automatic application of `aria-haspopup` and `aria-expanded` to the trigger
- Keyboard interaction
  - Enter or Space opens the menu
  - Escape closes the menu and returns focus to the trigger
  - Arrow keys move between menu items
- `role="menu"` applied to the container
