## Accessibility

The `nys-modal` component includes the following accessibility-focused features:

- Trap focus inside the modal so keyboard users can’t tab to background content.
- Return focus to the triggering element when the modal closes.
- Support keyboard interaction — allow Esc to close the modal.
- Announce the modal to screen readers using role="dialog" or role="alertdialog" and aria-modal="true".
- Give the modal a descriptive title (aria-labelledby or heading element).
- Prevent background scroll so users don’t lose context.

## Events

The `nys-modal` component emits the following events:

1. `nys-click`: Emitted when the modal is clicked.
2. `nys-focus`: Emitted when the modal receives focus.
3. `nys-blur`: Emitted when the modal loses focus.

**Event details** \
The event’s `detail` object includes:
- id (string): The id of the modal.
- checked (boolean): Whether the modal is currently checked.

You can listen to these events using JavaScript:

```js
// Select the modal component
const modal = document.querySelector("nys-modal");

// Listen for the 'nys-click' event
modal.addEventListener("nys-click", (event) => {
  const {id, checked} = event.detail;
	console.log("Modal (${id}) is clicked. Checked: ${checked}");
});

// Listen for the 'nys-focus' event
modal.addEventListener("nys-focus", () => {
	console.log("Modal is focused");
});

// Listen for the 'nys-blur' event
modal.addEventListener("nys-blur", () => {
	console.log("Modal lost focus");
});
```
